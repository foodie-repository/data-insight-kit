# Interview Questions

도메인 전문가에게 물어볼 질문 후보를 정리한다. 질문은 한 번에 하나의 결정을 좁혀야 한다.

kit은 확인 단계마다 기본 질문(행의 의미·핵심 대상·컬럼 의미·제외 기준·지표
기준·비교 축·판단 범위·용어)을 부족한 항목 우선으로 자동으로 묻는다
(interview loop v2). 이 파일에는 **그 도메인에서만 나오는 특화 질문**을 적는다
— 기본 질문의 중복이 아니라 보완이다. 단계 구분은 파이프라인 정지점과 같다:
데이터 확인 → 분석 방향 → (조건부) 1차 결과 확인 → 대시보드 구성 → 보고서 구성.

## 시작 질문

- 이 분석 결과로 어떤 판단이나 다음 행동을 결정하시나요?
- 분석 대상과 제외할 대상은 무엇인가요?
- 어떤 기간, 세그먼트, 지역, 제품군을 우선 봐야 하나요?

## 데이터 확인 후 질문

- 샘플과 컬럼 해석이 도메인 관점에서 맞나요?
- 결측이나 제외된 범위가 분석 결론에 영향을 주나요?
- 추가로 붙여야 할 기준 데이터가 있나요?

## 분석 전략 확인 질문

- 추천 KPI와 비교 기준이 실제 업무 판단에 맞나요?
- 더 중요한 세그먼트나 예외 케이스가 있나요?
- 보고서에서 피해야 할 표현이나 과장된 결론이 있나요?

## 1차 결과 검토 질문 (조건부 단계)

- 1차 발견이 업무 상식과 맞나요? 낯선 결과가 있다면 데이터 특성 때문인가요?
- 결론 수위를 낮춰야 하는 항목이 있나요?
- 이 결과로 말하면 안 되는 것(원인·성과·추천)이 있나요?

## 보고서 작성 전 질문

- 보고서 독자는 누구이고, 어떤 용어·표현 수준이 맞나요?
- 공개 가능한 범위와 내부용으로 남길 내용의 경계는 어디인가요?
- 결론에서 반드시 피해야 할 문구가 있나요?
