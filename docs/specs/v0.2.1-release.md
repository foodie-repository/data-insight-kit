# data-insight-kit v0.2.1 release

상태: 승인됨 — 사용자가 v0.2.1 릴리스 실행을 요청함(2026-07-18)

## 1. 목적

병합된 두 변경을 로컬 작업 결과가 아니라 Codex와 Claude Code 사용자가 같은 제품
본체로 설치할 수 있는 v0.2.1 배포본으로 고정한다.

- checkpoint 게이트 무결성 패치(Codex 교차검증 반영): manifest 자기서명 정책
  우회 제거 + checkpoint 상태 파일 직접 위조 차단 + 우회 negative 회귀 테스트.
- domain-pack 저작 문서 보강: 저작 워크플로 계약, 안티할루시네이션 용어 열,
  흩어진 회사 문서용 3층 구조.

이번 release는 새 분석 기능을 추가하지 않는다. 병합된 core의 버전·manifest
정합성, 결정적 배포, 원격 설치 검증만 수행한다. 블라인드 사용자 UAT와 실제
domain/statistical route smoke는 후속 작업이다.

## 2. 배포 단위와 버전 권한

- 배포 단위는 하나의 `data-insight-kit` root다. Claude Code와 Codex는 별도 kit가
  아니라 같은 root의 얇은 adapter를 읽는다.
- v0.2.1의 제품 버전은 `pyproject.toml`, `.claude-plugin/plugin.json`,
  `.codex-plugin/plugin.json`에서 모두 `0.2.1`이어야 한다.
- `.claude-plugin/marketplace.json`에는 version을 중복 기록하지 않는다.
- `pyproject.toml`은 `package = false`인 실행 환경 정의지만, 설치 화면과 진단에서
  혼동이 없도록 제품 release 버전과 맞춘다.

## 3. Codex와 Claude Code manifest 계약

- Codex manifest는 현재 plugin validator를 통과해야 한다. 시스템 validator helper가
  설치 중 갱신 등으로 제공되지 않으면, 격리된 `CODEX_HOME`에서 현재 Codex CLI의
  marketplace add와 plugin add를 통과하는 것을 런타임 검증으로 사용하고 helper
  부재를 기록한다.
- `.codex-plugin/plugin.json`에는 validator가 거부하는 `hooks` 필드를 넣지 않는다.
  checkpoint hook은 기본 발견 위치 `.codex/hooks.json`에 두고 실제 설치 smoke에서
  발견 여부를 확인한다.
- Claude Code manifest와 marketplace는 strict validation을 통과해야 한다.
- 두 adapter는 `window.openai`, proprietary asset, 외부 plugin cache, 개인 skill에
  의존하지 않는다.

## 4. 배포 트리 생성 계약

- 제품 단일 원천은 상위 저장소의 추적된 `data-insight-kit/` tree다.
- 배포 tree는 source release commit의 `data-insight-kit/` subtree를 `git archive`로
  추출한다. 로컬 파일을 임의 복사해 배포본을 만들지 않는다.
- 최신 `vk-dist/main`을 기준으로 일반 commit과 fast-forward push를 사용한다.
  force push는 하지 않는다.
- `runs/*`, `.env`, `connectors/.env`, `.venv`, cache, Playwright 작업물, 비공개
  원천 데이터는 배포 tree에 들어가면 안 된다.
- standalone clone에서도 같은 경계가 유지되도록 `.gitignore`는 `.venv/`,
  `.pytest_cache/`, `.ruff_cache/`, `.playwright-cli/`, `.DS_Store`를 직접 제외한다.
- `tests/`와 release 검증 문서는 사용자 설치와 회귀 확인을 위해 배포본에 포함한다.

## 5. 검증 순서

1. source branch에서 adapter/release 집중 테스트와 전체 pytest를 통과한다.
   fresh clone의 입력 없는 `_install-check --dry-run`도 stage 명령을 출력해야 하며,
   실제 실행에서만 source를 필수로 요구한다.
2. Codex plugin validator와 Claude Code strict validator를 통과한다.
3. 추적 subtree로 배포 worktree를 만들고 source tree와 파일·내용 hash가 같은지
   확인한다.
4. 배포 worktree에서 전체 pytest와 core `--dry-run`을 통과한다.
5. `vk-dist/main`에 fast-forward push한다.
6. 원격 GitHub 저장소를 새 격리 설정에 등록해 Codex와 Claude Code plugin을
   설치하고 skill, hook, core dry-run을 확인한다.
7. 원격 설치 smoke가 green일 때만 배포 저장소 HEAD에 `v0.2.1` annotated tag를
   만들고 push한다.

## 6. 설치 명령 계약

공개 문서는 로컬 경로 예시와 함께 원격 저장소 설치 명령을 제공한다.

Codex:

```bash
codex plugin marketplace add foodie-repository/data-insight-kit --ref v0.2.1
codex plugin add data-insight-kit@data-insight-kit
```

Claude Code:

```bash
claude plugin marketplace add foodie-repository/data-insight-kit
claude plugin install data-insight-kit@data-insight-kit
```

## 7. 완료 조건과 후속 경계

v0.2.1 release 완료 조건은 source merge, 세 버전 일치, 두 manifest 검증, source와
distribution tree 일치, 배포 worktree 테스트, 원격 Codex/Claude 설치 smoke,
`v0.2.1` tag다.

다음 항목은 이번 release의 대리 승인 대상이 아니다.

- 처음 보는 사용자가 수행하는 블라인드 UAT
- 실제 domain mode checkpoint 답변과 end-to-end smoke
- 실제 statistical route dependency 승인과 결과 검토
- predictive/causal 기능 확대
