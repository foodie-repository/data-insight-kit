## 발견 (v2 smoke)

`docs/dashboard-design-system.md`의 프로필 3종은 사용자가 제공한 태블로
대시보드 샘플 유형을 반영해 설계됐고, 렌더러(`templates/dashboard.html`)에
CSS·JS 분기(renderAnalystPanel/renderOperationsPanel)가 구현돼 있다. 그러나
실렌더 관측(sbiz statistical run, analyst_workspace) 결과 **프로필 간 차이가
사용자에게 체감되지 않는다** — 아키타입의 시그니처 요소가 구현에 없어서
어떤 프로필이든 "흰 카드 + 탭" 한 가지 문법으로 보인다.

## 관측된 갭 (샘플 대비)

| 아키타입 시그니처 (태블로 샘플) | 현재 렌더러 | 심각도 |
|---|---|---|
| KPI 타일: 값 + 스파크라인 + 전기 대비 ▲▼% | 값+라벨만 (플랫) | 높음 — 3 프로필 공통 체감 요소 |
| 사이드 레일 내비 (operations형) | 탭만 | 중간 |
| 스몰 멀티플 (반복 지표 비교) | 없음 | 중간 |
| 코호트/매트릭스 표 (셀 배경 그라데이션) | 플레인 표 | 중간 |
| 도넛/게이지 (구성·달성률) | 없음 | 낮음 (남용 위험, 선택적) |
| analyst 첫 화면 밀도 6~8패널 | 첫 탭 차트 1개 (탭 분산) | 높음 |
| 진단 배지·결론형 헤드라인 히어로 | story strip으로 부분 존재 | 낮음 |

## 사용자 제공 샘플 아키타입 요약 (2026-07-12 재첨부분)

1. **마이크로파이낸스(Unity Reach)형** — 좌측 메트릭 컬럼 + 중앙 차트 그리드,
   도넛 게이지(성별), 강조색 1색 원칙.
2. **Mobile App Performance형(analyst)** — KPI 4타일(vs 이전 기간 델타),
   도넛+연령 막대, 코호트 리텐션 매트릭스(셀 그라데이션), 좌측 레일.
3. **Contact Center형(operations)** — 상단 KPI+스파크라인 3연속, 좌 사이드바,
   고객별 스몰 멀티플 카드 페이지네이션.
4. **Executive Performance형(executive)** — KPI 4타일(스파크 영역+YoY),
   지표 전환 탭, 기준선 있는 막대.
5. **Quota Attainment형** — 같은 지표의 다형 표현(게이지/불릿/표) 카탈로그.
6. **Edinburgh Operational형(operations)** — 좌 브랜드 레일, SLA 상태색,
   연도별 타깃 상태 막대, 상세 표.
7. **Support Cases형(analyst/operations)** — 시간대×요일 히트맵, 프라이어리티
   막대+스파크, 소형 멀티플, 기간 비교 헤더.
